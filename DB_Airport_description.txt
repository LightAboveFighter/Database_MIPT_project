

Проект - сайт для просмотра/брони билетов и регистрации на рейсы.

Сущности и их свойства:
  Пользователь - аккаунт на сайте, через который исполняется заявленный функционал проекта.
    Набор билетов - массив билетов, которые пользователь забронировал
    Учетные данные - обязательные данные для регистрации (адрес почты, логин, пароль)

  Самолет - структура, отвечающая за самолет, хранящая номер борта и список билетов отвечающих определенным местам - их количество тоже хранится в структуре.
    Номер борта
    Набор билетов
    Количество мест
    Компания владелец

  Рейс - структура, отвечающая за полет самолета с пассажирами. Для доступности требует от пользователя регистрацию в билете.
    Номер рейса
    Дата и время
    Номер борта исполняющего полет
    Список билетов
    Точки пути


Пользователь входит на сайт под своим паролем и именем, и имеет возможность просматривать доступные рейсы, бронировать их, и оформлять регистрацию.

Каждому пользователю может приналдежать несколько билетов.
Каждый пользователь - учетные данные и набор билетов.
Каждый билет содержит номер рейса и номер пользователя.
Каждый рейс - набор билетов, пункты назначения, дата и время, и, конечно же, номер самолета.
Каждый самолет отвечает за несколько рейсов, которые обновляются по мере исполнения им этих рейсов.
Каждый самолет - набор мест в определенном количестве, каждому из которых может принадлежать билет.

В случае задержки рейса, всем пользователям, назначенным на этот рейс, отправляется уведомление о сдвиге рейса, параметры вылета меняются.


Данный проект служит помощью для организации аэропортам при организации взаимодействия пользователей с системой заказой билетов, при этом одновременно
осуществляя учет данных рейсов и самолетов.

Набор пользователей <-> набор рейсов <-> набор самолетов - необходимые составляющие, для работы с данным продуктом.

Описания процессов:
  Пользователь покупает билет - создается структура "билет" из номера пользователя и рейса, у самолета, привязанному к рейсу, билетом занимается выбранное место.
  Задержка рейса - всем пользователям, привязанным к рейсу (рейс -> самолет -> билет) отправляется уведомление о изменениях в графике, далее изменяются
  данные рейса - дата и время, а также данные регистрации у билета.
  Пользователь регистрируется на рейс - меняются данные регистрации у билета.
  Пользователь просматривает билеты - ему показываются все рейсы из бд, отсортированные по доступности(не прошедшие), дате/времени, цене и т.д
  Назначение даты/времени рейса - изменяется поле рейса "дата/время"
